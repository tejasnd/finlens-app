# FinLens — Demo Script

A ~3-minute walkthrough that shows the full stack (React + FastAPI), the RAG
pipeline, and MCP — the three things worth showing a hiring manager.

## Setup (once)

```bash
# Terminal 1 — backend
cd backend && source .venv/bin/activate
ollama pull llama3.2:3b            # or llama3.1 for sharper answers
uvicorn app.main:app --reload --port 8000

# Terminal 2 — frontend
npm run dev                        # http://localhost:5173
```

Upload a CSV (a sample is at [src/__tests__/fixtures/sample-transactions.csv](../src/__tests__/fixtures/sample-transactions.csv)) and complete couple setup with two names.

## Part 1 — Full-stack + couple budgeting (45s)
- Show the **Overview**, **Couple** (settlement / who-owes-whom), and **Monthly** tabs.
- Point out the network tab: categorization hits **`POST /api/categorize`** on the FastAPI backend, not the browser.

## Part 2 — RAG natural-language querying (60s)
- Open the **Ask** tab.
- Ask: *"How much did we spend on dining out, and where?"*
- Narrate the pipeline while it answers:
  - transactions embedded **on-device** (sentence-transformers),
  - top-k retrieved by cosine similarity,
  - retrieved rows **+ exact SQL aggregates** injected into the prompt (hybrid — so totals are correct),
  - answer generated by the **local** Ollama model.
- Expand "Retrieved N relevant transactions" to show the retrieval is real.

## Part 3 — MCP agent (60s)
- In the backend terminal:
  ```bash
  python mcp_servers/agent_host.py "What did we spend the most on, and are we settled up?"
  ```
- Narrate: a custom MCP **host** spawns two MCP **servers** (`finlens-mcp`,
  `gmail-bills-mcp`) over stdio, advertises their tools to the LLM, and runs a
  tool-calling loop. Watch the `· called <tool>(...)` lines on stderr.
- (Optional) Show the same servers in **Claude Desktop** using the config in
  [backend/mcp_servers/README.md](../backend/mcp_servers/README.md), then ask
  *"Find my latest credit-card bill and compare it to what we spent."* — this is
  the Gmail (`search_bills` → `get_bill`) + finance (`get_spend_by_category`)
  composition. Requires the one-time Gmail OAuth setup.

## Talking points (for the inevitable questions)
- **"Why hybrid RAG?"** Pure vector retrieval can't sum hundreds of rows, so
  aggregation is done in SQL while retrieval supplies relevant context.
- **"Why local embeddings / Ollama?"** Privacy — financial data and embeddings
  never leave the machine; cloud LLM is an opt-in fallback with your own key.
- **"Why author MCP servers instead of just calling an API?"** MCP lets an LLM
  agent dynamically choose tools and compose across domains (finances + Gmail);
  the server is reusable by any MCP host, including Claude Desktop.
- **"Why no vector DB?"** N is small (a few thousand rows); brute-force cosine is
  sub-millisecond, so a vector DB would be unjustified complexity.
